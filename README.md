# 현장체험학습 안전 관리 앱

현장체험학습에서 학생 안전을 지키기 위해 동분서주하시는 선생님들께 조금이나마 도움이 되었으면 하는 마음으로 만든 웹 앱입니다.  
수십 명의 학생을 인솔하며 점호를 챙기고, 사건사고에 대응하고, 학부모와 소통해야 하는 선생님들의 수고를 덜어드릴 수 있기를 바랍니다.

> 이 프로젝트는 양현고등학교의 현장체험학습을 위해 개발되었지만, 어느 학교에서나 자유롭게 활용할 수 있도록 공개합니다.  
> 아래 설명을 따라 Firebase와 Vercel 계정만 있으면 **무료**로 운영할 수 있습니다.

---

## 주요 기능

| 대상 | 기능 |
|------|------|
| **관리자** | 학생·교사 계정 관리, 점호 세션 생성/마감, 전체 현황 대시보드, 버스/숙소/일정 데이터 관리 |
| **교사** | 실시간 점호 진행, 승차 현황 모니터링, 학생 검색, 사건사고 등록, 점호 이력 조회 및 CSV 내보내기 |
| **학생** | QR 코드로 자가 체크인, 일정·숙소·비상연락처 조회, 분실물 신고 |
| **학부모** | 로그인 없이 자녀의 점호 현황 조회 |

**세부 기능**
- 실시간 점호 — QR 스캔 또는 교사 직접 탭 체크인, 마감 5분 전 미응답 학생 푸시 알림
- 호차별 탑승률 배너 — 승차점호 진행 중 호차별 진행률 실시간 표시
- 분실물 게시판 — 분실·습득 신고, 상태 변경(습득 → 반환 완료)
- 공지 채팅 — 관리자→교사→학생 일방향 공지 채널
- 학부모 조회 포털 — 학번 입력만으로 자녀 점호 현황 확인 (개인정보 최소화)

## 기술 스택

- **프론트엔드**: Next.js 14 (App Router), TypeScript, Tailwind CSS, shadcn/ui
- **데이터베이스**: Firebase Firestore (실시간 구독)
- **인증**: Firebase Authentication (이메일/비밀번호)
- **푸시 알림**: Firebase Cloud Messaging (FCM)
- **배포**: Vercel (자동 CI/CD, 무료 플랜 사용 가능)
- **PWA**: 홈 화면 추가, 오프라인 대응

### 준비물

- [GitHub](https://github.com) 계정
- [Firebase](https://firebase.google.com) 계정 (무료)

### 전체 흐름

```
1. 이 저장소를 Fork
2. Firebase 프로젝트 생성 및 설정
3. Vercel에 연결하여 배포
4. 관리자 계정 설정
5. 엑셀 파일로 학생 데이터 일괄 업로드
```

## 1단계: 저장소 Fork

1. 이 페이지 우상단 **Fork** 버튼 클릭
2. 본인 GitHub 계정에 저장소가 복사됩니다
3. 이후 모든 설정은 복사된 본인 저장소를 기준으로 합니다

---

## 2단계: Firebase 프로젝트 설정

### 2-1. 프로젝트 생성

1. [Firebase Console](https://console.firebase.google.com) 접속
2. **프로젝트 추가** → 프로젝트 이름 입력 (예: `our-school-trip-2026`)
3. Google Analytics는 선택 사항

### 2-2. 웹 앱 등록

1. 프로젝트 홈 → **웹 앱 추가** (`</>` 아이콘)
2. 앱 닉네임 입력 후 **앱 등록**
3. 표시되는 `firebaseConfig` 값 메모 (환경 변수에서 사용)

```js
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "your-project.firebaseapp.com",
  projectId: "your-project",
  storageBucket: "your-project.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abc123",
};
```

### 2-3. Authentication 활성화

Firebase Console → **Authentication** → **시작하기** → **이메일/비밀번호** 활성화

### 2-4. Firestore 데이터베이스 생성

Firebase Console → **Firestore Database** → **데이터베이스 만들기**  
→ **프로덕션 모드** → 리전: `asia-northeast3` (서울)

### 2-5. 푸시 알림(FCM) 설정

1. Firebase Console → **프로젝트 설정** → **클라우드 메시징** 탭
2. **웹 푸시 인증서** → **키 쌍 생성**
3. 생성된 **VAPID 공개 키** 메모

### 2-6. 서비스 계정 키 발급

1. Firebase Console → **프로젝트 설정** → **서비스 계정** 탭
2. **새 비공개 키 생성** → JSON 파일 다운로드
3. 이 파일 내용이 `FIREBASE_SERVICE_ACCOUNT_JSON` 환경 변수가 됩니다

> **중요**: 이 JSON 파일은 절대 GitHub에 커밋하지 마세요. `.gitignore`에 이미 포함되어 있습니다.

## 3단계: Firestore 보안 규칙 배포

### Firebase CLI 설치 및 배포

```bash
npm install -g firebase-tools
firebase login
firebase use --add   # 목록에서 생성한 프로젝트 선택
firebase deploy --only firestore:rules
```

> **주의**: 이 명령은 Vercel 배포와 **별개**입니다. 보안 규칙을 변경할 때마다 실행해야 합니다.

---

## 4단계: Vercel 배포

### 4-1. Vercel 프로젝트 생성

1. [Vercel](https://vercel.com) → **Add New Project**
2. Fork한 본인 GitHub 저장소 선택
3. Framework: **Next.js** (자동 감지)
4. **Deploy** 클릭 (환경 변수는 다음 단계에서 추가)

### 4-2. 환경 변수 설정

Vercel 프로젝트 → **Settings** → **Environment Variables**에 아래 변수를 모두 추가합니다.

**클라이언트용 변수 (NEXT_PUBLIC_ 접두사)**

| 변수명 | 값 |
|--------|----|
| `NEXT_PUBLIC_FIREBASE_API_KEY` | firebaseConfig의 apiKey |
| `NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN` | firebaseConfig의 authDomain |
| `NEXT_PUBLIC_FIREBASE_PROJECT_ID` | firebaseConfig의 projectId |
| `NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET` | firebaseConfig의 storageBucket |
| `NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID` | firebaseConfig의 messagingSenderId |
| `NEXT_PUBLIC_FIREBASE_APP_ID` | firebaseConfig의 appId |
| `NEXT_PUBLIC_VAPID_KEY` | FCM 웹 푸시 VAPID 공개 키 |

**서버용 변수**

| 변수명 | 값 |
|--------|----|
| `FIREBASE_SERVICE_ACCOUNT_JSON` | 서비스 계정 JSON 파일 내용 전체 (한 줄) |

### 4-3. 재배포

환경 변수 추가 후 **Deployments** → **Redeploy**

---

## 5단계: 최초 설정 (관리자 계정)

1. 배포된 URL 접속 → **회원가입** (교사 가입 코드는 초기에 아무 값이나 입력)
2. Firebase Console → **Authentication**에서 가입 계정의 **UID** 복사
3. Firebase Console → **Firestore** → `users` 컬렉션 → 해당 UID 문서 → `role` 필드를 `admin`으로 변경
4. Firebase Console → **Firestore** → `settings` 컬렉션 → `global` 문서 생성:

```
teacherSignupCode  :  (교사들에게 알려줄 가입 코드 문자열)
studentLoginEnabled: true
```

## 엑셀 파일 형식

파일은 **하나의 .xlsx 파일**에 아래 **5개 시트**가 모두 포함되어야 합니다.  
각 시트의 **1행은 반드시 헤더(컬럼명)**이어야 합니다.

### 시트 이름 규칙

시트 이름은 아래 중 하나를 사용하면 자동 인식됩니다 (대소문자 구분).

| 시트 | 인식되는 이름 |
|------|--------------|
| 학생 명단 | `students`, `1_students`, `학생`, `학생명단`, `학생목록`, `Students` |
| 숙소 호실 | `rooms`, `2_rooms`, `호실`, `호실정보`, `객실`, `Rooms` |
| 여행 일정 | `schedule`, `3_schedule`, `일정`, `일정표`, `스케줄`, `Schedule` |
| 버스 정보 | `buses`, `4_buses`, `버스`, `버스정보`, `호차`, `Buses` |
| 비상 연락처 | `contacts`, `5_contacts`, `연락처`, `비상연락처`, `긴급연락처`, `Contacts` |

---

### 시트 1: 학생 명단 (`students`)

| 컬럼명 | 필수 | 형식 | 설명 |
|--------|------|------|------|
| `이름` | ✅ | 문자 | 학생 이름 |
| `학년` | ✅ | 숫자 | 학년 (예: 2) |
| `반` | ✅ | 숫자 | 반 번호 (예: 3) |
| `번호` | ✅ | 숫자 | 출석 번호 (예: 15) |
| `호차` | ✅ | 숫자 | 배정 버스 호차 (예: 2) |
| `호실` | 선택 | 문자 | 숙소 호실 (예: 201호) |
| `층` | 선택 | 숫자 | 숙소 층수 |
| `학생연락처` | 선택 | 문자 | 학생 휴대폰 번호 |
| `보호자연락처` | 선택 | 문자 | 학부모 휴대폰 번호 |
| `건강요주의사항` | 선택 | 문자 | 알레르기, 지병 등 |
| `특이사항` | 선택 | 문자 | 기타 메모 |
| `잔류여부` | 선택 | Y/N | 학교 잔류 학생 표시 |
| `요양호여부` | 선택 | Y/N | 특별 관리 대상 |
| `비행편` | 선택 | 문자 | 항공편 정보 (해외 여행 시) |

**예시 데이터:**

| 이름 | 학년 | 반 | 번호 | 호차 | 호실 | 학생연락처 | 보호자연락처 |
|------|------|-----|------|------|------|-----------|------------|
| 홍길동 | 2 | 1 | 1 | 1 | 201호 | 010-1111-2222 | 010-3333-4444 |
| 김영희 | 2 | 1 | 2 | 1 | 201호 | 010-5555-6666 | 010-7777-8888 |
| 이철수 | 2 | 2 | 1 | 2 | 305호 | 010-9999-0000 | 010-1234-5678 |

| `층` | 선택 | 숫자 | 층수 |
| `담당교사` | 선택 | 문자 | 담당 교사 이름 |
| `정원` | 선택 | 숫자 | 수용 인원 |
| `특이사항` | 선택 | 문자 | 메모 |

**예시 데이터:**

| 호실 | 층 | 담당교사 | 정원 |

### 시트 3: 여행 일정 (`schedule`)

| 컬럼명 | 필수 | 형식 | 설명 |
|--------|------|------|------|
| `일차` | ✅ | 숫자 또는 "1일차" | 여행 일차 (1, 2, 3...) |
| `일정명` | ✅ | 문자 | 일정 이름 |
| `시작시각` | 선택 | HH:MM | 시작 시간 |
| `종료시각` | 선택 | HH:MM | 종료 시간 |
| `장소` | 선택 | 문자 | 장소명 |
| `점호유형` | 선택 | 문자 | `정시점호` 또는 `승차점호` |
| `비고` | 선택 | 문자 | 추가 안내 |

> **일차 컬럼명**은 `일차`, `Day`, `날짜`, `일`, `차수`, `일자`, `구분`, `day` 중 하나를 사용하면 자동 인식됩니다.  
> **시작시각 컬럼명**은 `시작시각`, `시작`, `시작시간`, `Start`, `시작 시각` 중 하나 사용 가능합니다.

**예시 데이터:**

| 일차 | 시작시각 | 종료시각 | 일정명 | 장소 | 점호유형 |
|------|---------|---------|--------|------|---------|
| 1 | 07:00 | 07:30 | 학교 집결 | 학교 정문 | 정시점호 |
| 1 | 08:00 | 09:30 | 버스 이동 | 고속도로 | 승차점호 |
| 1 | 10:00 | 12:00 | 경복궁 관람 | 경복궁 | |
| 1 | 19:00 | 20:00 | 숙소 도착 후 휴식 | 호텔 | 정시점호 |
| 2 | 08:00 | 09:00 | 조식 후 출발 | 호텔 | 승차점호 |

### 시트 4: 버스 정보 (`buses`)

| 컬럼명 | 필수 | 형식 | 설명 |
|--------|------|------|------|
| `호차` | ✅ | 숫자 | 버스 번호 (1, 2, 3...) |
| `탑승반` | 선택 | 문자 | 탑승 반 (예: 1반,2반) |
| `인솔교사1` | 선택 | 문자 | 인솔 교사 이름 |
| `인솔교사2` | 선택 | 문자 | 보조 인솔 교사 이름 |
| `기사명` | 선택 | 문자 | 버스 기사 이름 |

**예시 데이터:**

| 호차 | 탑승반 | 인솔교사1 | 인솔교사2 | 기사명 | 기사연락처 |
|------|--------|---------|---------|--------|-----------|
| 1 | 1반,2반 | 김선생 | 이선생 | 홍기사 | 010-1111-1111 |
| 2 | 3반,4반 | 박선생 | 최선생 | 강기사 | 010-2222-2222 |
| 3 | 5반,6반 | 정선생 | | 조기사 | 010-3333-3333 |

### 시트 5: 비상 연락처 (`contacts`)

| 컬럼명 | 필수 | 형식 | 설명 |
|--------|------|------|------|
| `이름` | ✅ | 문자 | 연락처 이름 |
| `구분` | 선택 | 문자 | 분류 (예: 병원, 경찰, 학교) |
| `연락처` | 선택 | 문자 | 전화번호 |

| 경찰 | 관할 경찰서 | 032-111-0000 | Y |
| 숙소 | 호텔 프런트 | 032-222-0000 | Y |
| 학교 | 교무실 | 031-333-0000 | N |
| 인솔 | 단장 (교감) | 010-9999-9999 | N |

### 엑셀 파일 주의사항

- 파일 형식: `.xlsx` 또는 `.xls`
- 1행: 컬럼 헤더 (위 표의 `컬럼명` 그대로)
- 2행부터: 실제 데이터
- 숫자 칸에 빈칸이 있어도 괜찮습니다 (0으로 처리됨)
- 시간은 `07:00` 형식으로 입력하거나, 엑셀 시간 형식으로 입력해도 자동 변환됩니다
- 학생 데이터는 **재업로드해도 기존 로그인 정보(uid)가 유지**됩니다
- 일정·연락처는 재업로드 시 기존 데이터가 **모두 교체**됩니다

## 자주 묻는 질문

**Q. 교사가 비밀번호를 잊어버렸어요.**  
A. 로그인 화면 → **비밀번호 찾기**로 이메일 재설정 링크를 받습니다.  
관리자가 대신 처리: `/admin` → **교사** 탭 → 해당 교사 → **비밀번호 초기화** 버튼.

**Q. 학생이 QR을 스캔했는데 체크인이 안 돼요.**  
A. 진행 중인 점호 세션이 있어야 합니다. 세션이 없으면 체크인이 불가합니다.

**Q. 푸시 알림이 오지 않아요.**  
A. 앱 접속 시 브라우저의 알림 권한을 **허용**해야 합니다.  
iOS Safari에서는 **홈 화면에 추가(PWA)** 한 경우에만 푸시가 동작합니다.

**Q. 엑셀 업로드 후 일정이 이상하게 들어가요.**  
A. 업로드 페이지의 **일정 점호 유형 일괄 설정** 버튼을 눌러 점호 유형을 자동 적용하세요.

**Q. 보안 규칙을 수정했는데 반영이 안 돼요.**  
A. `firebase deploy --only firestore:rules` 명령으로 따로 배포해야 합니다.
